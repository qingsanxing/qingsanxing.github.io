单元测试

测试

Junit是什么

Junit是Java语言编写的第三方单元测试框架名称

单元测试概念

  • 单元：在Java中，一个类就是一个单元
  • 单元测试：程序猿编写的一小段代码，用来对某个类中的某个方法进行功能测试或业务逻辑测试。

Junit单元测试框架的作用

Junit单元测试框架的使用步骤

测试方法注意事项

单元测试 是对方法逐一测试， 对方法有一些要求：

  • 必须是public修饰的，没有返回值，没有参数
  • 必须使注解@Test修饰

如何运行测试方法

  • 选中方法名 –> 右键 –> Run ‘测试方法名’ 运行选中的测试方法
  • 选中测试类类名 –> 右键 –> Run ‘测试类类名’ 运行测试类中所有测试方法
  • 选中模块名 –> 右键 –> Run ‘All Tests’ 运行模块中的所有测试类的所有测试方法

如何查看测试结果

  • 绿色：表示测试通过
  • 红色：表示测试失败，有问题

Junit常用注解(Junit4.xxxx版本)

  • @Before：用来修饰方法，该方法会在每一个测试方法执行之前执行一次。
  • @After：用来修饰方法，该方法会在每一个测试方法执行之后执行一次。
  • @BeforeClass：用来静态修饰方法，该方法会在所有测试方法之前执行一次。
  • @AfterClass：用来静态修饰方法，该方法会在所有测试方法之后执行一次。

Xml概述

  • 概述
    • extensive markup language : 可扩展标签语言
    • 简单说就是指xml文件中的标签可以任意定义。
  • dom树

  • 代码实现

Xml组成

  • 组成
    • 文档声明
    • 属性
    • 元素
    • 注释
  • 文档声明
  • 属性
  • 元素
  • 注释
    • 和html注释一样!

Xml约束概述

  • 概述
    • Xml文件中，元素名称、文本内容、属性名称、属性值都可以随意定义。如果要对这些进行规范，那么就需要用到Xml约束
  • 分类
    • dtd约束
      • 简单约束，只能对元素名称、属性名称进行约束
    • schema约束
      • 复杂约束，对对元素名称、属性名称、文本内容、属性值进行约束

dtd约束)

  • 开发步骤
    • ①定义xml文件
    • ②定义dtd文件
    • ③dtd文件关联xml文件
  • ①定义xml文件
  • ②定义dtd文件
  • ③dtd文件关联xml文件(内部关联、外部关联)
  • 注意事项
    • dtd约束只能对元素名称、属性名称进行约束。

Schema约束

  • 开发步骤
    • ①定义xml文件
    • ②定义schema文件
    • ③schema文件关联xml文件
  • ①定义xml文件
  • ②定义schema文件
  • ③schema文件关联xml文件

dom4j解析(掌握)

  • 概述
    • 一个简单灵活开放源代码的解析xml类库，xml解析就是对xml文件进行读写操作
    • 在很多开源的框架中大量使用dom4j，例如: mybatis、sping
  • 常用方法
    • attributeValue(String attributeName)
      • 获取指定属性值
    • attribute(String attributeName)
      • 获取指定属性对象
    • Attribute.getValue()
      • 获取属性值
    • elementText(String elementName)
      • 获取指定元素文本内容
    • element(String elementName)
      • 获取指定元素对象
    • Element.getText()
      • 获取文本内容
  • 开发步骤
    • ①引入jar包
    • ②使用dom4j解析xml文件
      • 2.1,获取解析对象
      • 2.2,获取XML文档对象
      • 2.3,获取根节点对象
      • 2.4,获取每个节点信息
  • ①引入jar包
  • ②使用dom4j解析xml文件
