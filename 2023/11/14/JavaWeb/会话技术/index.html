会话技术概述

  • 为什么？
    • 之前javaweb中存储数据使用ServletRequest、ServletContext两个域对象。
    • ServletRequest域对象范围太小了；ServletContext域对象范围太大了。
  • 概述
    • 会话指的是，打开浏览器，发起请求，会话开始；关闭浏览器，会话结束。
  • 作用
    • 用于存储数据，并实现数据的共享。
  • 分类
    • cookie
      • 浏览器端会话技术，数据保存在浏览器端
    • session
      • 服务器端会话技术，数据保存在服务器端

cookie概述

  • 概述
    • cookie是一个浏览器端的缓存文件，用于保存客户访问浏览器的一些信息；它也是http请求头(Cookie)和http响应头(Set-Cookie)
  • 作用
    • 可以在cookie缓存文件中，保存一些客户访问的数据，通过http请求头(Cookie)和http响应头(Set-Cookie)实现数据共享。
  • 属性
    • name : cookie名称
    • value : cookie值
    • path : cookie访问路径
    • maxAge ： cookie存活时长

cookie执行流程

  • 执行流程
    • ①浏览器请求Demo01Servlet，创建Cookie对象，并通过Set-Cookie响应头将Cookie数据携带回浏览器，并保存cookie缓存文件中
    • ②当浏览器再次发起请求时，会通过Cookie请求头携带cookie缓存文件中的信息到服务器

cookie相关设置

  • 名称限制(setValue)
  • 存活时长限制(setMaxAge)
    • -1 : 默认值，会话关闭cookie就会销毁
    • 0 : cookie立即销毁
    • 正数 : cookie存活时长，以秒为单位
  • 访问路径限制(setPath)
    • 设置访问哪些资源时需要携带cookie数据，默认值是”/项目名称”
    • 比如：path=”/项目名称/目录1”，只要uri前面是”/项目名称/目录1”,就可以携带cookie信息
    • 比如：path=”/项目名称/目录1/资源1”，uri必须完全匹配path，才会携带cookie信息

session概述

  • 概述
    • 服务器端的会话技术；
    • 在浏览器端保存一个特殊标识(jsessionid)，数据保存在服务器端的。每次请求，浏览器都会携带这个特殊标识到服务器，从而找到对应的数据。
    • 也是一个域对象(ServletRequst、HttpSession、ServletContext)。
  • 域对象
    • ServletRequest
      • 数据在当前请求中被共享
    • HttpSession
      • 数据在当前会话(一次会话可以有多次请求)中被共享
    • ServletContext
      • 数据在当前项目中被共享

  • ①往session中存储数据

session执行流程

  • 执行流程
    • ①第一次请求Demo08Servlet时，会创建session对象(CC159DF439E658D29C396E189F48D54B)，会在内存中保存该session对象
    • ②通过Set-Cookie响应头将(jessionid=CC159DF439E658D29C396E189F48D54B)携带回浏览器保存
    • ③第二次请求Demo09Servlet时，通过Cookie请求头携带(jessionid=CC159DF439E658D29C396E189F48D54B)数据到服务器
    • ④服务器根据jessionid在内存中找对应的session对象，如果有直接使用，如果没有就创建

session相关配置

  • 生命周期
    • 重启服务器，session会销毁。
    • 重启浏览器，session不会销毁，只是会将浏览器端的jsessionid销毁，所以会创建一个新的session对象。
    • 在tomcat的web.xml中，设置session的默认存活时长为30分钟。
  • session销毁
    • invalidate()方法
